PROMPT 1: Create Initial Database Schema with 7 Core Tables
────────────────────────────────────────────────────────────

Create your first MySQL database called "hospital_OLTP_system"

Create a SQL file called "create_schema.sql" that sets up these 7 base tables:

1. departments - Hospital departments (Cardiology, Neurology, etc)
   - Fields: department_id (primary key), name, phone, email
   
2. doctors - Medical doctors
   - Fields: doctor_id, first_name, last_name, specialization, license_number
   
3. patients - Patient records
   - Fields: patient_id, first_name, last_name, date_of_birth, phone
   
4. appointments - Doctor appointments
   - Fields: appointment_id, patient_id (FK), doctor_id (FK), appointment_date
   
5. medical_records - Patient medical history
   - Fields: record_id, patient_id (FK), diagnosis, treatment_date
   
6. insurance_companies - Insurance providers
   - Fields: company_id, company_name, phone, email
   
7. billing - Patient billing records
   - Fields: billing_id, patient_id (FK), amount, billing_date

Include PRIMARY KEYs for each table and setup FOREIGN KEY relationships
between tables that reference each other (like patient_id in appointments).

PROMPT 9: Implement Data Load Ordering Logic
──────────────────────────────────────────────

Update the Python scripts to load data in the correct order:

The order matters because some tables reference others through foreign keys.
Always load "parent" tables before "child" tables.

LOAD ORDER:
1. Reference tables first (icd_codes, cpt_codes) - these have no FK dependencies
2. Organizational tables (departments, facilities, rooms, beds, equipment)
3. Staff tables (doctors, nurses, staff, specialists)
4. Patient tables (patients, addresses, contacts, allergies)
5. Transactional tables (appointments, encounters, lab orders, etc)
6. Financial tables (insurance, invoices, payments)

This prevents FK constraint violations during data loading.
